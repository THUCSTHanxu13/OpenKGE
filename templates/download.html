Toolkits

Fast-TransX

There is an Efficient implementation of TransE and its extended models for Knowledge Representation Learning Edit, including TransH, TransR, TransD, TranSparse and PTransE. The overall framework has underlying design changes for acceleration and supports multi-threaded training. [download]

Tensorflow-TransX

There is an Efficient implementation based on TensorFlow for knowledge representation learning (KRL), including TransE, TransH, TransR and TransD. We use C++ to implement some underlying operations such as data preprocessing and negative sampling. For each specific model, it is implemented by TensorFlow with Python interfaces so that there is a convenient platform to run models on GPUs. [download]

Wikidata

Embeddings of the relations

The embeddings for each relation(property) in the Wikidata. The data are in a binary format with one embedding per line. For each line, there are numbers of consecutive float to represent its embedding. [data]

Embeddings of the entities

The embeddings for each entity(item) in the Wikidata. The data are in a binary format with one embedding per line. For each line, there are numbers of consecutive float to represent its embedding. [data]

Dumps of the Wikidata

Here are the dumps of the Wikidata. If necessary, you can click Wikidata for more information. [data]

Relation2id

The mapping from relaitons(property) of the Wikidata to their corresponding serial numbers in a format with one mapping per line. For each line, there is a relation and its serial number split by a tab. [data]

Entity2id

The mapping from entities(item) of the Wikidata to their corresponding serial numbers in a format with one mapping per line. For each line, there is an entity and its serial number split by a tab. [data]
